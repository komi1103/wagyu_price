
# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？

* リモートリポジトリ：ネット上に配置して複数人で共有するためのリポジトリ
* ローカルリポジトリ：開発者一人ひとりが使用するために自分のPCに配置するためのリポジトリ

## プッシュとマージの違いは何でしょうか？

+ 変更内容の反映先が違う。
  - push：ローカルの変更内容をリモートの同じブランチに反映させる。
  - merge：ローカルの変更内容をローカルにある違うブランチ（masterなど）に反映させる。

## コミットとプッシュの違い

+ 保存するリポジトリが違う。
  - commit：変更内容をローカルリポジトリのブランチに保存する。
  -push：コミットした内容をリモートリポジトリの同じブランチに反映させる。

## コミットのメッセージはどのように書いてあげるのが最適でしょうか？
* 履歴を細かく見なくても、メッセージだけで変更内容が分かるように書くのが良い。

## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？
* プルリクエストを作成することで、変更内容を第三者に見てもらうこと（コードレビュー）ができるため、事前にバグを発見することができる。
* Pullしなければ、ローカルのmasterブランチに変更内容は保存されない。

## コンフリクトを起こしてしまった場合、どう対処すべきですか？
+ 対処方法として３つ
  - 先にマージされた変更内容を取り込む
  - 後にマージしようとしている変更内容を取り込む
  - どちらの変更内容も取り込む

* ソースコードを書いた人と相談しながら作業を進めることが大切。
